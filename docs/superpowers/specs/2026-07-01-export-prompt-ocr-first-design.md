# Export 프롬프트: 원문 확보(OCR) 먼저 → 분석 2단계 (spec)

날짜: 2026-07-01
관련: ADR-007(앱은 분석 안 함, 외부 AI 위임), ADR-008/009(Export·버전된 고정 템플릿), `src/lib/prompt.ts`

## Context

현재 Export 프롬프트(`prompt.ts`의 "## 너에게(지시)")는 "각 사진을 OCR해 연결하라. 그런 다음 [주제/반복/관계/글감/인터뷰/독서노트]"를 **한 문단에 섞어** 지시한다. 외부 AI가 OCR을 건너뛰거나 대충 하고 바로 분석으로 넘어가면 근거 없는 결과가 나온다.

사용자 요구: **OCR로 원문(전사본)을 먼저 확보·정리한 뒤** 나머지 작업을 진행해야 한다. 이 **기초자료 확보(추출)는 코어** — 앞으로 분석 프롬프트를 다양하게 바꿔도 항상 먼저 도는 안정적 토대다.

## 결정

프롬프트 지시를 **명시적 2단계**로 재구성한다. 두 단계를 헤더로 분리해, 나중에 2단계(분석)를 갈아끼워도 1단계(원문 확보) 코어는 불변이 되게 한다.

- **1단계 — 원문 확보(핵심·항상):** 첨부 사진(책 페이지)을 각각 **OCR → 정리된 전사본**으로 만들고 `capture-NN`별로 **먼저 출력**한다. 오탈자·줄바꿈·잘린 글자를 정리하고, 각 캡처의 담은 글(passage)·내 생각(note)과 묶어 "정돈된 원문" 블록으로. 사진 없는 캡처는 담은 글/내 생각을 그대로 정리. → 이 산출물이 이후 모든 작업의 **기초자료**.
- **2단계 — 분석(가변):** 1단계의 정리된 원문을 **근거로** 주제/반복/관계/글감 후보/인터뷰 질문/독서노트 초안을 만든다.

## 컴포넌트/변경 (`src/lib/prompt.ts`)

- `md` 템플릿의 "## 너에게 (지시)" 섹션을 아래 구조로 교체:
  - 짧은 맥락 1–2문장(기존 유지) + **명확한 순서 강제**:
    - `### 1단계 — 원문 정리 (먼저)`: "첨부 사진은 책 페이지다. **먼저 각 사진을 OCR**해 `capture-NN`별 **정돈된 전사본**을 출력하라(오탈자/줄바꿈 정리, 담은 글·내 생각과 묶어서). 사진 없는 캡처는 담은 글/내 생각을 그대로 정리하라. 이 전사본이 이후 작업의 기초자료다."
    - `### 2단계 — 분석 (그 다음)`: "위 정리된 원문을 근거로: 1) 주요 주제 2) 반복/강조 3) 캡처 간 관계 4) 글감 후보 5) 인터뷰 질문 6) 독서노트 초안."
  - "먼저 전사본을 출력"을 명시(중간 산출물 강제) → 근거·안정성.
- `PROMPT_TEMPLATE_VERSION` `"v1" → "v2"`(의미 변경 반영). 템플릿 표기 줄(`capture-prompt vN`)도 자동 반영.
- 태그 범례·캡처 데이터 블록·태그 분포 섹션·파일 수집(capture-NN.jpg)·`ExportPackage` 구조는 **불변**.
- 마이크로카피: plain·일관. "원문"/"전사본"/"기초자료" 용어 일관.

## 제약/디자인
- 앱은 분석하지 않는다(ADR-007) — 프롬프트 텍스트만 바뀐다. 코드 로직/데이터 모델 변경 없음.
- 사진이 0개인 Export도 자연스럽게(1단계는 "사진 없으면 담은 글/내 생각 정리"로 성립).
- 문구는 한국어(기존 프롬프트와 동일 톤).

## 검증
테스트 프레임워크 없음 → `npm run build` + preview로 prompt.md 미리보기 확인 + `npm run test:pdf`(프롬프트가 PDF에도 들어가므로 회귀 없음).
1. `npm run build` 무에러.
2. preview Export: prompt.md 미리보기에 1단계(원문 정리 먼저)·2단계(분석) 헤더가 순서대로, 버전 `v2` 표기.
3. 사진 있는 캡처 / 사진 없는 캡처 / 혼합 각각에서 문구가 자연스러운지.
4. `test:pdf` PASS(프롬프트 텍스트 렌더 회귀 없음).

## 미해결/주의
- 2단계(분석) 작업 목록은 지금 6개 유지 — 향후 목적별(독서노트/블로그/인터뷰) 가변은 별건(ADR-009 Phase 2). 이번엔 1·2단계 분리 구조만 확립.
- 외부 AI가 실제로 순서를 지키는지는 프롬프트 설계로 유도할 뿐 보장 불가 — "먼저 전사본 출력" 명시가 최선의 유도.
