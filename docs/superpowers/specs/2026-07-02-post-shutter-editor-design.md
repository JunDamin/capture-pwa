# 촬영 후 편집 화면 — 사진 경로 재설계 (spec)

날짜: 2026-07-02
관련: ADR-011(캡처 예산 — appMs 기준), ADR-013(iOS), ADR-014(캡처 모델), §7(태그 우선 배치), design-language.md
선행: 책 전환+입력 UX 배치 머지 완료(main).

## Context

현재 사진 경로: 셔터 → 동결 화면 위 태그 탭 → note 바텀시트 → 저장 → 카메라 복귀. 문제:
- 동결+시트 구조라 **찍힌(크롭된) 결과를 제대로 못 봄** — 흔들림/잘림을 저장 후에야 발견.
- 시트에는 note만 — 사진 캡처에서 담은 글·페이지를 못 적음.
- 입력모드와 흐름이 달라 학습 이중화.

사용자 결정: **셔터 직후(태그 전) 편집 화면으로 전환**, 입력모드와 거의 동일하게 진행, **다시 찍기** 제공.

## 새 사진 흐름 — 대형 편집 시트 (속도 우려 반영: 화면 전환 아님)

**셔터 → [크롭+압축을 시트 상승 애니메이션과 병행] → 대형 편집 시트 상승(화면 ~90%, 카메라가 위에 살짝 보임): 크롭 결과 사진 + 다시 찍기 + 태그 → 담은 글 → 생각 → 페이지 → 저장 → 시트 하강, 라이브 복귀.**

- **속도:** 화면 전환/재마운트 없음 — **카메라가 밑에 계속 라이브**. 시트 상승(~0.3s 스프링, 기존 note 시트와 동일) 동안 크롭·압축(~0.1-0.2s) 병행 → 체감 지연 0. **다시 찍기 = 시트 하강 한 번, 즉시 재촬영**.
- 탭 수 동일(태그 + 저장 = 2탭) — **3초 루프 의미 유지**(ADR-011 appMs 회귀 금지, 계측 확인). CLAUDE.md "사진 경로" 기술 **개정**(ADR-018).
- **다시 찍기**: 시트 상단(사진 옆) — 탭 시 결과 폐기(입력값 초기화) → 시트 하강 → 라이브.
- 카메라 화면 단순화: 뷰파인더 + 크롭 프레임 + 셔터 + 모드 토글 + 뒤로가기. **동결 상태·태그 행·기존 note 시트·photo-ctrl 제거**(§9 동결 프레임 유지도 자연 대체 — 동결 소멸). 시트가 열린 동안 셔터/프레임은 시트 뒤(비활성).

## 컴포넌트/변경 (capture.ts 중심)

### 1. 셔터 핸들러
- 기존 freeze 캔버스 + cover 매핑 크롭은 유지하되, **셔터 시점에 `cropResizeCompress`까지 실행**해 `pendingPhoto: { blob, width, height }` 확보 → objectURL로 편집 화면에 표시. (압축 ~100-200ms — 전환 애니메이션과 병행, 완료 전엔 사진 자리에 freeze 캔버스 축소 표시 또는 즉시 blob 대기 — 구현 단순화 우선: await 후 전환해도 체감 무해하면 그대로.)
- 실패(카메라/크롭 에러) 시: 기존처럼 사진 없이 편집 화면 진입(텍스트 캡처 가능) 또는 라이브 복귀 + 토스트 — **사진 없이 편집 화면 진입 채택**(생각만 저장 가능해졌으므로 일관).

### 2. 편집 시트(= 입력 패널 구성 재사용 + 사진 헤더, 대형 바텀시트)
- 형태: 기존 `.sheet` 패턴 확장한 **대형 시트**(높이 ~90%, 상단 라운드, 밝은 톤) — 카메라 위로 상승. 내부 스크롤(`overflow-y`, 키보드 대응).
- 구성(§7 순서와 통일): **사진 헤더**(크롭 결과 `<img>` objectURL — **반응형 축소**: 최대 높이 ~30vh·패딩 최소, 탭하면 openImageViewer로 크게) + **"다시 찍기"**(≥44px) → 태그행 → 담은 글 → 생각 → 페이지 → 저장.
- 입력모드 패널과 마크업/스타일 최대 공유(같은 필드 클래스 재사용 — 단 입력모드는 화면 고정 패널, 시트는 상승 컨테이너: 구조 공유 범위는 구현자가 판단하되 필드·검증·토스트 로직은 공용).
- 저장: `rec.image = pendingPhoto.blob` + passage/note/page/tag → addCapture(기존 로직) → objectURL revoke → 시트 하강 + "저장했어요" 토스트.
- 검증: 사진 있으면 태그만 필수(사진이 콘텐츠, passage/note 선택). 사진 없이 시트가 뜬 경우(크롭 실패 폴백)는 입력모드 규칙(passage‖note + 태그).
- 책 전환 pill(이전 배치)은 시트가 열린 동안에도 상단 pill로 동작하게 할지 — **v1은 시트 안에서는 미지원**(시트 닫고 입력모드에서 가능; 단순화). 가드 조건은 기존 `currentMode !== "input"` 유지.

### 3. 카메라 화면 정리
- photo 모드에서 태그 행(`photo-ctrl .tagrow`)·note 시트(`sheet`/`scrim`)·동결(`is-frozen`) 관련 UI/상태 제거(또는 미사용화). phase 상태기계 단순화: `live → editing → live`.
- 크롭 프레임은 라이브에서만(동결 소멸로 §9 규칙 정리).
- 예산 계측(budget): shutterMs(셔터→편집 표시), saveMs 유지 — HUD 의미 재정의 최소.

### 4. detail.ts 반응형 사진 (같은 원칙)
- 기존 상세 화면 사진도 **반응형 축소 + 패딩 최소**(최대 높이 제한, 탭하면 크게) — 편집 화면과 동일 규칙/CSS 공유.

### 5. 문서
- **ADR-018**: 사진 경로 재설계(셔터→편집 화면, 태그는 편집 화면에서; 탭 수 불변으로 3초 루프 의미 유지; 동결/시트 제거). CLAUDE.md의 "사진 모드 3초 루프" 서술 갱신.

## 제약/디자인
- 셔터→편집 전환 즉시성(appMs 회귀 금지 — 계측으로 확인). 이미지는 `<img>`+objectURL만(ADR-013).
- 편집 화면은 밝은 톤(입력모드와 동일) — 카메라(다크)와 명확한 전환.
- 마이크로카피: "다시 찍기", "저장"(기존 액션명 유지).
- objectURL 수명: pendingPhoto 표시 URL은 저장/다시 찍기/화면 이탈 시 revoke.

`npm run build` + `test:pdf` + preview(데스크톱: 셔터→편집 표시·다시 찍기·저장·검증 규칙·책 전환 pill) + **iOS 실기기**(전환 체감 속도, 크롭 결과 표시, 예산 HUD).
